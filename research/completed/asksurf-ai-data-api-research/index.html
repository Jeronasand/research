1. 执行摘要

产品定位
AskSurf / Surf 是 AI-powered crypto intelligence 平台，面向开发者、AI Agent、投资研究、链上分析。
主要能力
Chat API、Data API、Onchain SQL、Data Catalog、CLI、Skill、MCP Server。
主要数据类型
链上数据、市场行情、交易所订单簿、社交信号、新闻、钱包、DeFi、预测市场。
最适合集成的路径：先用 REST Data API 和 Onchain SQL 做最小验证，再用 Chat API 做自然语言研究层，最后按需接入 CLI / Skill / MCP 到 Agent 工作流。

5. API 使用规范

鉴权

统一使用：

Authorization: Bearer <SURF_API_KEY>

不要把 Key 写死在前端或仓库里。建议由后端代理请求，前端只请求自己的业务后端。

分页

列表接口通常使用：

limit=20&offset=0

常见最大 limit 为 100。批量同步应做分页循环和重试。

链名规范

使用完整 canonical chain name：

ethereum
solana
base
arbitrum
polygon

避免使用 eth、sol、matic 这类缩写。

错误处理

需要处理：

400 参数错误
401 鉴权失败
402 余额/付费相关，具体语义需以返回体为准
404 资源不存在
422 参数校验失败
429 速率限制
502 上游/网关错误
速率限制具体阈值没有完全公开。集成时需要按 429 做指数退避，并记录 credits_used、余额和失败响应体。

6. 如何使用 API

6.1 Chat API：curl

curl --request POST \
  --url https://api.asksurf.ai/gateway/v1/chat/completions \
  --header "Authorization: Bearer $SURF_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "model": "surf-1.5",
    "messages": [
      {"role": "user", "content": "Summarize BTC market today with key drivers."}
    ],
    "stream": false,
    "reasoning_effort": "medium"
  }'

6.2 Chat API：JavaScript fetch

async function askSurf(message) {
  const res = await fetch("https://api.asksurf.ai/gateway/v1/chat/completions", {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${process.env.SURF_API_KEY}`,
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      model: "surf-1.5",
      messages: [{ role: "user", content: message }],
      stream: false
    })
  });

  if (!res.ok) throw new Error(`Surf API error ${res.status}: ${await res.text()}`);
  return await res.json();
}

6.3 Chat API：Python

import os
import requests

url = "https://api.asksurf.ai/gateway/v1/chat/completions"
headers = {
    "Authorization": f"Bearer {os.environ['SURF_API_KEY']}",
    "Content-Type": "application/json",
}
payload = {
    "model": "surf-1.5",
    "messages": [{"role": "user", "content": "Summarize ETH market today."}],
    "stream": False,
}

r = requests.post(url, headers=headers, json=payload, timeout=120)
r.raise_for_status()
print(r.json())

8. 风险、限制与未确认项

项目状态影响
底层 Chat 模型供应商未披露不能假设一定是 OpenAI；OpenAI-compatible 只代表接口兼容
Web Search 底层供应商未披露无法确认是自建索引、第三方搜索 API，还是混合方案
新闻源完整列表未披露新闻可信度和覆盖范围需要实际调用后评估
速率限制精确阈值未披露生产环境必须处理 429 和重试退避
Data API 路径漂移存在可能文档中 endpoint 数量和路径写法存在差异，必须以 OpenAPI 为准
主站前端调用接口未完整确认需要浏览器 DevTools Network 抓包才能准确确认主站实时调用
工程建议：不要把 HTML 报告里的示例路径视为永久稳定接口。真正接入前，应自动拉取 https://api.asksurf.ai/gateway/openapi.json，生成 SDK 或类型定义，并把接口变更检测加入 CI。
