计费 / 限制 / 成本控制

Dune 采用 usage-based credit 系统。查询消耗与实际计算资源相关，而不是简单按 API 请求次数计费。

Credits 消耗因素

  • 查询复杂度。
  • 扫描数据量。
  • 选择的 Engine：small / medium / large。
  • Join 数量和 Join 的表大小。
  • 读取结果大小。
  • Materialized View 刷新成本。
  • 失败执行也可能消耗费用。

已知限制/注意事项

项目说明
Small Engine默认共享集群；官方文档说明 2 分钟超时，最多 3 个并发执行。
结果大小官方文档提到 Query result 当前最大 32GB，超过会被截断，需要处理 partial result。
CSV Upload官方文档列出单次上传 500MB 限制；不同订阅层有不同存储上限。
Materialized View占用存储配额；免费层限制较小。
Billing失败执行也会计费；执行但不拉取结果，只收执行部分。

Agent 成本控制策略

  1. 默认小样本：先 LIMIT 10 验证字段。
  2. 默认短时间窗：先查 1 天、7 天，再扩大。
  3. 优先 curated table，不直接扫 raw logs。
  4. 重复任务使用 Materialized View。
  5. 对每个 Agent 任务设置 max_credit / max_time / max_rows。
  6. 定时同步 Usage API，超过阈值暂停自动执行。
  7. 危险操作需要人工确认：delete table、large engine、全历史扫描。
