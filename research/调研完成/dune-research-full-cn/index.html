执行摘要

Dune 已不再只是“做链上看板的社区工具”，而是演化为覆盖 Data Hub、Data API、Trino/BI Connector、Datashare、Sim、dbt Connector 与 AI Agents 的全栈链上数据平台。官方文档与官网当前口径显示，Dune 已索引 100+ 区块链、沉淀 3+ PB 数据、拥有 1.5M+ 数据集、6.5M+ 查询、200K+ 仪表盘，并被 1,500+ 付费客户 与 1M+ 用户 使用。[2][3]

从产品战略看，Dune 的核心优势不是某个单点功能，而是“一份多层链上数据仓 + 多种消费接口”：分析师可以在 Web 端写 SQL、做图表与公开分发；工程团队可通过 API、Trino、dbt 或 Datashare 把同一套数据接入生产环境；开发者则可用 Sim 做近实时钱包与 DeFi 场景；AI 工作流可通过 MCP、CLI 与 Skills 调用 Dune 数据。[4][5][11][12][17][18]

结论上，Dune 最适合三类需求：其一，跨链自定义研究与对外发布；其二，协议/交易平台/基金的历史数据仓与分析工程；其三，把链上数据嵌入产品与 AI 工作流。但若场景要求严格的毫秒级确定性、强标签交易信号或高度标准化的“财务可比指标”，Dune 往往需要与自建节点、Nansen、Token Terminal 等工具组合使用，而不应被视为单一万能解。[4][16][32][36]

公司与产品

Dune Analytics AS 成立于 2018 年，起源于挪威奥斯陆，由 Fredrik Haga 与 Mats Olsen 创立。About 页面在 2026 年显示公司已有 50+ 员工、累计融资约 8000 万美元；其使命表述为“让加密数据可访问”。[1] 官方叙事也清楚显示其路线变化：从早期面向分析师的社区化 SQL/仪表盘产品，逐步扩展为涵盖 企业数据仓、实时 API、AI 原生接口与链集成工具 的 onchain data platform。[1][25]

目标用户与使用场景

官方 use cases 将 Dune 明确分为 Analytics、Engineering、Accounting/Audit/AML 三大场景：分析团队用于跟踪生态、协议增长与市场趋势；工程团队用于构建数据管道、应用后端与机器学习/AI 派生分析；财务与合规团队则用于交易监控、账户对账、财务报告与市场操纵识别。[42][43][44]

简化判断：如果你的目标是“自己定义链上问题并把答案分发出去”，Dune 的产品-社区组合价值很高；如果你的目标是“把链上数据稳定嵌入产品或企业仓库”，Datashare、dbt Connector 与 Sim 才是更关键的购买理由。[4][16]

核心功能与技术架构

Dune 的底层查询能力来自 DuneSQL：官方文档说明其基于 Trino，以 Parquet 为底层列式存储，并通过 Delta Lake 管理版本与数据能力；产品比较页则将 Data Hub 架构标注为 OLAP, Delta parquet，典型查询速度约 3 秒。[4][10]

从体验上看，Dune 把同一份链上数据仓拆成多种消费接口：分析师走 Web SQL 和图表；BI 工具走 Trino；工程团队通过 API、dbt、Datashare；开发者通过 Sim；AI 通过 MCP/CLI/Skills。这种“一仓多口”设计，是其区别于单纯看板工具或单纯 API 供应商的核心。[4][11][12][17][18]

数据来源、质量与合规

数据来源与数据质量评估

官方数据目录把 Dune 数据分为三层：Raw（直接从节点索引的 block/transaction/log/trace）、Decoded（基于 ABI 解析后的合约事件与函数调用）、Curated（由 Dune 数据团队维护的跨链规范化数据集）；此外还有 Community / Partner Data，其更新节奏不受 Dune 控制。[6][7][8]

就刷新特征而言，官方文档显示：raw 层受出块频率与链重组影响；decoded 通常在 raw 入库后 15–60 秒 完成；curated 通常按 1 小时 刷新，部分 Solana 策展表为 6–7 小时；product comparison 页给出的更宏观口径为 EVM ~1–15 分钟、Solana ~1–5 分钟、Curated ~1 小时，而 Sim 则面向 <1 秒 的实时读接口。[4][7]

质量方面，Dune 的优势是：官方称 curated 数据经过自动质量检查，且方法学与限制对外文档化；Curated 层所依赖的开源转换仓 Spellbook 公开可见，可提升可审计性与可复核性。局限在于：raw 天生受链重组与节点稳定性影响；decoded 质量依赖 ABI、解析与覆盖；partner/community 数据不受 Dune 直接控制。学术综述也普遍指出，区块链数据系统存在标准化、存储引擎、可扩展性、完整性与可用性等系统性挑战。[8][9][31][40][41]

隐私与合规

Dune 位于挪威，并在条款中明确受 GDPR 约束。隐私政策显示，平台会处理姓名、用户名、邮箱、IP、浏览器及站内导航信息等个人数据；同时明确提示：由于 Dune 产品具有社交属性，免费服务中创建的内容会公开可见。[19][21]

对于客户上传到 Data Hub 的个人数据，DPA 将客户界定为 Data Controller、Dune 为 Data Processor；并特别强调如果涉及个人数据，应只在 非公开版本/私有仪表盘 中使用，公开暴露责任由客户与同团队用户承担。DPA 还披露了若干最低安全措施，包括 AWS 运行环境、MFA、环境隔离、集中日志、事件通知与审计配合。[20]

这意味着：若你的业务涉及 KYC、客户交易明细、员工或用户标识等敏感或可识别数据，Dune 更适合采用 Enterprise + 私有空间 / Datashare 方案，而不是直接把数据上传到公共内容流。另一个现实问题是，Trust Center 公开页只说明可自助申请下载合规材料和安全报告，并未在公开页面列出具体认证清单，因此若采购决策依赖某项特定认证，仍需销售或法务阶段逐项核验。[16][22]

定价、商业模式与竞争

定价与商业模式

Dune 的商业模式本质上是“免费入口 + 信用计费的数据计算平台 + 企业销售”。Data Hub 的公开套餐以 credits 为核心计费单位：Free 提供 2,500 credits；Analyst 为 $75/月（或年付折合 $65/月）并含 4,000 credits；Plus 为 $399/月（或年付折合 $349/月）并含 25,000 credits；Enterprise 为定制。官方文档强调 extra credits 与套餐内 credits 同价，且查询实际消耗取决于引擎、复杂度与实时资源用量。[13]

另一个价格信号是“导出与写入”：Billing 页显示 API/CSV 导出按数据量计费，Free / Analyst / Plus 分别为 20 / 10 / 2 credits 每 MB；写入操作会消耗 credits，Plus 存储上限为 15 GB，Queries Endpoint 仅在 Plus 与 Enterprise 开放。对 dbt Connector，官方写明它是 Enterprise-only 的使用量计费功能，而且“无单独平台费，只按 credits 付费”。[14][12]

优劣势分析

优势

平台完整度高。 从探索、图表、公开分发，到 API、BI、dbt、Datashare、实时 API 与 AI 接口，几乎覆盖了链上数据消费全生命周期。[4][17]

社区网络效应强。 200K+ 仪表盘、1.5M+ 数据集、6.5M+ 查询意味着内容发现与“先看别人怎么做”极具价值。[3]

透明度较好。 Curated 文档强调方法与限制公开，Spellbook 开源降低黑箱感。[8][31]

企业闭环明确。 Datashare 与私有空间满足“数据在你自己的仓库和安全边界里用”的需求。[16]

不足

信用计费的可预测性一般。 查询、导出、写入都会消耗 credits，且失败执行也计费，新团队需要建立成本守卫。[13][14]

“实时”是分层的。 Data Hub/Curated 并非全部秒级，真正近实时主要靠 Sim；历史分析与产品后端不能混为一谈。[4][7]

覆盖与口径不完全均一。 不同链、ABI、partner 数据、策展表的成熟度存在差异。[7][9]

部分企业信息公开度有限。 具体认证清单、Datashare/Sim 明确报价等公开页未完整披露。[22]

近期新闻、融资与结论

近期新闻与融资动态

近两年 Dune 的主线非常清晰：从“分析工具”走向“数据基础设施”。2024-11，Dune 收购实时链上仿真与分析公司 smlXL，强化实时能力；2025-05，Echo 正式更名为 Sim；2025-11，官方在 DuneCon25 宣布“成为 onchain data platform”，重点推出 dbt Connector、扩大 Datashare 到 BigQuery / Databricks，并强化 Sim；2026-03 至 2026-04，连续上线 MCP、CLI、Skills、OAuth、Visualization Management、Machine Payments Protocol，明显押注 AI/Agent 接口。[27][26][25][28][29][30]

融资方面，官方博客显示 Dune 于 2020-09 完成 $2M seed，2022-02 完成 $69.42M Series B；About 页面当前写法为累计融资约 $80M。截至 2026 年可见公开资料，未见官方披露新的后续融资轮次；第三方数据库 Tracxn 在 2026-04 的条目仍将最近一轮列为 2022-02-02 的 Series B。[23][24][1][39]

结论与建议

综合来看，Dune 当前最有竞争力的不是单一数据表，也不是单一 SQL 编辑器，而是“跨链数据仓 + 多接口消费 + 社区分发 + 企业闭环”的组合。对于需要自己定义指标、跨链验证假设、快速做仪表盘并公开或半公开传播的团队，Dune 仍是优先级很高的平台。[3][4]

如果你的目标是生产系统，建议按场景拆分采购：研究与增长分析 用 Data Hub / API；企业仓库集成 用 Datashare；链上产品后端 用 Sim；内部转化与私有模型 用 Enterprise + dbt Connector；AI 助手/自动化研究 则应优先验证 MCP / CLI / Skills。若你关注的是聪明钱标签与交易信号，可把 Dune 与 Nansen 组合；若关注财务可比与机构研究效率，可把 Dune 与 Token Terminal 组合。[12][16][18][32][36]

采购/落地建议：
  • 先把问题分类：探索分析、生产报表、实时后端、私有数据治理，不要用同一套餐硬覆盖全部需求。
  • 如果涉及 PII、客户账户或内部经营数据，优先评估 Enterprise、私有空间与 Datashare，并在上传前完成数据分级。
  • 建立 credit guardrail：为查询与月度 extra credits 设置上限，并为失败执行、导出与物化写入单独做预算。
  • 对关键指标保留“双轨校验”：Dune Curated 负责效率，自建核验或 raw 查询负责追责与审计。

开放问题与限制

本报告使用公开页面与文档进行整理，仍有三点需要注意：其一，Dune 部分价格页为动态渲染，公开抓取结果与 docs 个别字段存在轻微差异，本文优先采用官方 docs 的保守口径；其二，官网不同页面对链覆盖有“100+”与博客中“130+”等不同说法，本文统一采用更稳妥的“100+”；其三，Trust Center 未在公开页点名具体认证，因此相关合规要求应在采购尽调阶段二次核验。[2][3][22][30]
