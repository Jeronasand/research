DuneSQL 查询引擎

DuneSQL 是 Dune 的核心。它基于 Trino，并针对链上数据做了增强，例如 EVM 大整数、varbinary、链上地址/hash 处理、物化视图和大规模数据扫描。

关键能力

  • uint256：适合 EVM 大整数。
  • varbinary：适合 address、tx hash、event topic 等二进制链上字段。
  • Query Views：复用 SQL 逻辑。
  • Materialized Views：把查询结果存成可查询表。
  • 列式存储 + PB 级查询。

查询引擎大小

Engine特点适合
Small默认，共享 community cluster；官方文档提到 2 分钟超时、最多 3 个并发执行。小查询、测试、LIMIT 样本。
Medium更长 timeout，无并发限制；按实际计算资源消耗 credits。标准生产查询、Dashboard。
Large复杂任务，约为 Medium 的 2x capacity，速度更快但消耗更高。大 Join、重型分析、历史回溯。

Agent 写 SQL 的推荐规则

  • 所有查询默认加 LIMIT。
  • 所有时间序列表默认加 block_time > now() - interval 'N' day。
  • 优先使用 curated tables，例如 dex.trades、labels.addresses、cex.flows，不要直接扫 raw logs。
  • 多次复用的查询必须转为 Materialized View。
  • 跨大表 Join 前先做字段裁剪和时间过滤。
  • 查询失败时先缩小时间范围，不要直接切 Large。

示例：DEX 交易数据

SELECT
  blockchain,
  project,
  date_trunc('hour', block_time) AS hour,
  sum(amount_usd) AS volume_usd,
  count(*) AS trades
FROM dex.trades
WHERE block_time > now() - interval '1' day
GROUP BY 1,2,3
ORDER BY hour DESC
LIMIT 100;
