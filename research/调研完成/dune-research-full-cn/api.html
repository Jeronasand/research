Dune API 能力调研

Dune API 不是单一“数据读取接口”，而是围绕 Query 生命周期、结果读取、数据上传、物化视图、Pipeline、用量管理构建的一套数据平台 API。

API 能力总览

模块能力适合场景
Executions and Results执行 SQL、执行保存 Query、查状态、取结果、CSV/JSON 输出、取消执行。Agent 动态查询、后端服务定时拉指标、Dashboard 数据接口。
Query Management创建、读取、更新、归档、私有化 Query。把 Query 当作版本化资产管理。
Datasets列出、搜索、读取数据集元数据，按合约地址找 decoded datasets。AI Agent 自动找表、生成 SQL 前做 schema inspection。
Data UploadsCSV 上传、创建表、插入 NDJSON/CSV、清空、删除、列出上传表。把自有数据写入 Dune，与链上数据 Join。
Materialized Views创建/刷新/读取/列出/删除物化视图。构建可复用的中间指标层，降低重复查询成本。
Pipelines执行由多个 Query 和物化视图刷新组成的流程。复杂指标管道、日报/周报任务。
Usage查询 API 用量和 credit 消耗。成本控制、Agent 防失控。
WebhooksQuery 定时执行后推送结果到外部 URL。AI Pulse、Slack、飞书、手机推送。

典型 API 调用流程

方式 A：执行原始 SQL

POST https://api.dune.com/api/v1/sql/execute
Headers:
  X-Dune-Api-Key: YOUR_API_KEY
Body:
{
  "sql": "SELECT * FROM dex.trades WHERE block_time > now() - interval '1' day LIMIT 10",
  "performance": "medium"
}

适合 Agent 动态分析，但要注意成本和 SQL 安全。生产环境建议设置查询范围、LIMIT、时间窗口和成本上限。

方式 B：执行保存好的 Query

POST /api/v1/query/{query_id}/execute
GET  /api/v1/execution/{execution_id}/status
GET  /api/v1/execution/{execution_id}/results

适合你把稳定策略指标做成 Query 模板，然后由本地服务或 Codex 定时触发。

方式 C：直接读取最新结果

GET /api/v1/query/{query_id}/results

不会触发新的执行，但读取结果本身也可能按结果大小消耗 credits。适合读取已调度/已缓存结果。

API 设计要点

  • API Key 权限很关键：Agent 拿到 API Key 后，能做什么取决于 key scope。建议给 Agent 使用低权限 Key。
  • 执行与读取分离：执行 Query 生成 execution_id，然后再轮询状态和读取结果。
  • 支持 server-side 结果处理：结果读取侧支持分页、过滤、排序、采样，适合大结果集。
  • 结果大小有上限：官方文档提到 Query result 当前最大 32GB，超过会产生 partial result 场景。
  • 失败执行也可能收费：API billing 文档说明失败执行也会被计费，因此 Agent 必须做 SQL 预检查和成本约束。
