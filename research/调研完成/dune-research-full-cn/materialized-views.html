Materialized Views

Materialized View 是 Dune 内部构建数据管道的核心能力：把一个 Query 的结果存成表，后续 Query 可以直接读取这个结果，而不是每次重跑复杂 SQL。

适合使用 Materialized View 的情况

  • 同一个指标被多个 Query、Dashboard、Agent 反复使用。
  • 原始 Query 扫描数据量大、执行时间长。
  • 日报/小时报/告警任务需要稳定输出。
  • 需要把复杂多表 Join 拆成中间层。
  • 需要给 AI Agent 提供低成本、低风险的数据入口。

推荐分层

层级内容例子
Raw / CuratedDune 原始或官方整理表。dex.trades、cex.flows
MV Base过滤后的业务基础表。近 30 天目标 token DEX 交易。
MV Metric聚合指标。每小时成交量、买卖地址数、CEX 流入。
API Query面向系统/Agent 的轻量 Query。读取最新 signal 或 alert。

刷新策略

  • 分钟级：不推荐，Dune 不是低延迟行情源。
  • 小时级：适合 Dashboard、趋势监控、鲸鱼监控。
  • 每日：适合日报、研究报告、排名榜。
  • 事件触发：用 Pipeline 或外部调度触发。
