01

概要结论

Dune 的核心定位

Dune 是链上数据仓库与分析平台。它把多链原始数据、解码数据、社区查询、仪表盘、API、企业连接器与实时 API 组合成一套 Web3 数据基础设施。

适合你的使用方式

如果目标是“多数据源并行抓取 + 统一入库 + AI 记忆”，Dune 更适合作为链上结构化数据源和链上分析结果源，而不是完全替代自建事件监听器。

Data API执行 SQL、读取查询结果、管理 Query、上传数据、Pipeline、Webhook。
Sim API面向实时应用，提供钱包余额、交易、NFT、DeFi Position、Token Holder 等接口。
Datashare企业级数据流向 Snowflake、BigQuery、Databricks。
Trino / dbt通过 Trino endpoint 接入 BI、SQL Client、dbt 转换流程。
MCP / Skills面向 AI Agent，让 Agent 可查询数据集、执行查询、调用 Sim。

03

数据源整理

链上基础数据

交易、区块、事件日志、调用轨迹、地址活动、Token 转账、合约交互等。适合协议分析、资金流、交易量、用户留存、风险行为识别。

解码与标准化数据

将 ABI、合约事件、DEX/NFT/DeFi 等领域数据做结构化整理。适合减少自建 indexer 成本。

社区与公共查询

Dune 上已有大量公开 Query 与 Dashboard，可作为业务指标模板，也可通过 API 获取最新或历史执行结果。

自定义上传数据

通过 Data Uploads 创建表、插入数据、上传 CSV、清理或删除表。适合把内部标签、地址清单、映射表接入 DuneSQL。

实时钱包数据

Sim 提供余额、交易、活动、NFT、DeFi Position、Token Holder、Token Metadata 等实时接口，更偏应用后端。

企业数仓数据

通过 Datashare 或 Connector 把 Dune 数据带入 Snowflake、BigQuery、Databricks、PowerBI、Metabase、DBeaver 等环境。

Data API 能力表

分类 能力 典型端点 / 动作 说明
Execution 执行任意 SQL POST /api/v1/sql/execute 无需先创建 Query，适合动态分析和自动化工作流。
Execution 执行已保存 Query POST /api/v1/query/{query_id}/execute 可传参数与 performance tier，返回 execution_id。
Execution 查询执行状态 GET /api/v1/execution/{execution_id}/status 用于轮询 pending / executing / completed / failed 等状态。
Execution 获取执行结果 GET /api/v1/execution/{execution_id}/results JSON 结果，可配合分页、过滤、排序、采样。
Execution 获取 CSV 结果 GET /api/v1/execution/{execution_id}/results/csv 适合导出、离线处理、简单 ETL。
Execution 获取最新 Query 结果 GET /api/v1/query/{query_id}/results 不触发新执行，适合 Dashboard 缓存读取。
Query 创建 / 更新 / 读取 Query POST /query、PATCH /query 后端可程序化维护 Dune 查询资产。
Query 私有化 / 归档 Query Private、Archive、Unarchive 适合团队权限治理和查询生命周期管理。
Data 上传自定义表 Create Table、Insert Data、Upload CSV 把内部地址标签、项目映射、黑名单等加入 DuneSQL。
Data Dataset 搜索 List / Get / Search Datasets 让系统或 Agent 自动发现可查询数据表。
Data Materialized View Upsert、Refresh、List、Delete 对高成本查询做预计算，提高读取性能。
Ops Pipeline Execute Pipeline、Get Pipeline Status 用于多步骤查询或有依赖关系的数据任务。
Ops Usage / Billing Get Usage 监控 API 消耗，避免 Agent 或自动任务滥用额度。
Ops Webhook Create Webhook 适合任务完成通知、结果更新通知和自动触发下游处理。
05

Sim 实时 API

Sim 是 Dune 的实时链上应用 API，更适合钱包、Portfolio、交易监控、风控、DeFi 聚合展示，而不是离线分析。

核心接口能力

  • Balances：地址的多链 Token 余额与 USD 价值。
  • Transactions：地址相关交易记录。
  • Activity：从交易中派生的转账、Swap、Approve 等活动。
  • Collectibles：ERC721 / ERC1155 NFT 持仓与基础元数据。
  • DeFi Positions：借贷、LP、收益策略、Vault 等 DeFi 持仓。
  • Token Holders：ERC20 Holder 列表与余额排序。
  • Token Info：Token 元数据、价格、供应量、Logo 等。
  • Stablecoins：稳定币与收益型稳定币余额。

适用判断

如果你要做“用户输入钱包地址后立即返回资产 / NFT / 活动 / DeFi 仓位”，优先使用 Sim。如果你要做“某协议过去 180 天交易量、用户画像、跨协议对比”，优先使用 Data API + DuneSQL。

连接器与 Datashare

Dune Data Warehouse
Trino Endpoint
trino.api.dune.com
BI / dbt / Data Warehouse

Trino Connector

只读访问 Dune 数据仓库，适合 PowerBI、Tableau、Metabase、DBeaver、Hex、Notebook、内部 SQL 分析环境。

dbt Connector

通过 dbt-trino 在 Dune 上跑生产级转换流程，支持增量模型、测试、CI/CD。通常需要企业权限与 Data Transformations。

Datashare

把 Dune 数据流入 Snowflake、BigQuery、Databricks。适合企业已有数仓体系，不希望通过 API 分页反复拉取大规模数据。

MCP / CLI / Agent Skills

Dune MCP

MCP 让 Claude、Cursor、Codex、OpenClaw 等兼容 Agent 可以直接搜索表、执行查询、生成可视化或读取结果。适合把 Dune 接成“链上数据工具”。

Dune Skills

Dune 官方提供 Agent Skills 仓库，包含 dune 和 sim 两类 Skill：前者偏 Dune CLI、数据集、Query、Usage；后者偏实时钱包和 Token 查询。

API 使用规范

认证

  • Dune Data API：使用 X-Dune-API-Key 或文档中对应 header。
  • Sim API：使用 X-Sim-Api-Key。
  • 不要把 API Key 放在前端；必须经由你的后端代理。
  • 按服务商、环境、任务类型拆 Key，方便额度与权限控制。

调度建议

  • 低频指标：用定时任务获取 latest query result。
  • 高频实时：用 Sim，设置后端缓存与降级。
  • 大结果集：分页拉取，落到自己的数据库。
  • AI Agent：加用量阈值、白名单 Query、只读默认权限。
Data API：执行任意 SQL
curl -X POST "https://api.dune.com/api/v1/sql/execute" \
  -H "Content-Type: application/json" \
  -H "X-Dune-Api-Key: YOUR_DUNE_API_KEY" \
  -d '{
    "sql": "SELECT * FROM dex.trades WHERE block_time > now() - interval '\''1'\'' day LIMIT 10",
    "performance": "medium"
  }'
Data API：执行已保存 Query
curl -X POST "https://api.dune.com/api/v1/query/YOUR_QUERY_ID/execute" \
  -H "X-Dune-API-Key: YOUR_DUNE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query_parameters": {"address": "0xd8da6bf26964af9d7eed9e03e53415d37aa96045"},
    "performance": "large"
  }'
Sim API：读取 NFT Collectibles
curl --request GET \
  --url https://api.sim.dune.com/v1/evm/collectibles/0xd8da6bf26964af9d7eed9e03e53415d37aa96045 \
  --header 'X-Sim-Api-Key: YOUR_SIM_API_KEY'

Provider 接口建议

把 Dune 作为一个 Provider。Go 内部定义统一接口：Fetch(ctx, task) ([]NormalizedRecord, error)。Data API 和 Sim API 分别实现不同 Adapter。

AI 记忆建议

不要直接把原始 API 响应塞入记忆。先转成统一结构：来源、链、地址、指标、时间、查询 SQL、置信度、过期时间、原始链接。

选型建议

需求 推荐模块 原因
协议指标、交易量、留存、链上报表 Data Hub + Data API 先写 Query 验证，再用 API 自动化读取。
钱包资产、NFT、DeFi 仓位实时展示 Sim 接口语义更贴近应用，避免自己建索引器。
企业级 BI / 数仓融合 Datashare / Trino Connector 减少 API 分页拉取和重复数据搬运。
生产级数据建模和转换 dbt Connector 适合团队维护模型、测试、增量、CI/CD。
AI Agent 自动查链上数据 MCP + Skills + 白名单 Query 让 Agent 使用工具，不直接暴露任意高成本查询。

风险与限制

费用与额度执行查询、读取结果、Sim 实时调用都可能产生费用或额度消耗。必须在后端做限流和缓存。
结果大小大结果集要分页。Dune 文档提到超大结果可能被截断，需要显式允许 partial results。
结果保留执行结果存在过期时间；长期数据应落库到自己的数据库。
权限隔离Agent 不应拿全权限 API Key。按只读、执行、上传、企业连接器做隔离。
实时 vs 分析Data API 不适合直接承担高并发实时钱包接口；Sim 更适合这个场景。
企业功能门槛Datashare、dbt Connector、部分高阶能力可能需要 Enterprise 或额外开通。
