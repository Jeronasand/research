01

Dune / API

Dune / API

Data API

Dune 的通用数据接口，用于执行 SQL、执行已保存 Query、读取结果和管理数据资产。

详细解释
Data API 更偏离线分析、自动化报表和数据管道。它的核心是把 DuneSQL 查询能力通过 HTTP 暴露出来，让后端服务可以程序化触发查询、轮询状态、读取 JSON 或 CSV 结果。
适合用法
适合协议指标、历史数据分析、Dashboard 结果同步、定时 ETL、AI Agent 白名单查询。
注意点
不适合直接承担高并发钱包资产接口；实时资产展示优先看 Sim。

Dune / API

Sim API

Dune 的实时链上应用 API，面向钱包资产、交易、NFT、DeFi 持仓和 Token 数据。

详细解释
Sim API 与 Data API 的定位不同。Sim 更接近应用后端可直接调用的语义化接口，隐藏了大量链上索引与聚合逻辑。
适合用法
适合 Portfolio、钱包首页、地址画像、Token Holder、NFT 持仓、DeFi Position 展示。
注意点
不要用 Sim 替代复杂历史 SQL 分析；复杂聚合仍应使用 DuneSQL / Data API。

02

查询语言

查询语言

DuneSQL

Dune 基于 Trino 生态的 SQL 查询语言，用来查询链上数据表和上传数据表。

详细解释
DuneSQL 用 SQL 方式访问多链数据、解码表、社区数据集和用户上传表。对分析师友好，也便于把查询沉淀成可复用 Query。
适合用法
适合写协议收入、DEX 交易量、地址留存、资金流、Token 分布等链上指标。
注意点
复杂查询需要关注性能、扫描量、分页和结果大小。

03

Dune / 资产

Dune / 资产

Query

Dune 上保存的一段 SQL 查询，可以公开、私有、归档或通过 API 执行。

详细解释
Query 是 Dune 中最核心的可复用分析资产。一个 Query 可以有参数、执行历史和结果缓存，也可以被 Dashboard 或 API 使用。
适合用法
适合把经过验证的 SQL 固化为稳定数据接口。
注意点
不要让 Agent 任意生成高成本 Query；生产环境建议白名单 Query。

Dune / 数据发现

Dune / 数据发现

Dataset

Dune 中可搜索、可查询的数据表或数据集合。

详细解释
Dataset 是 Agent 或数据服务进行自动发现时的重要对象。通过 Dataset 搜索可以找到可查询表、字段和元信息。
适合用法
适合自动发现可用表，或者为 Agent 提供数据目录。

06

API / 执行流程

API / 执行流程

Execution ID

一次查询执行任务的唯一标识，用于轮询状态和获取结果。

详细解释
调用执行 SQL 或执行已保存 Query 后，通常先拿到 execution_id，再通过状态接口确认任务完成，最后读取结果。
适合用法
适合异步任务、调度系统、Webhook 或后端轮询。
注意点
不要假设请求发出后立即有结果；必须处理 pending、failed、cancelled 等状态。

数据工程

数据工程

Materialized View

物化视图，把高成本查询结果预计算并保存，后续读取更快。

详细解释
Materialized View 用于把昂贵 SQL 的中间结果或聚合结果固化，降低重复扫描成本。
适合用法
适合热点指标、日级聚合、复杂 Join 或多 Agent 共用结果。
注意点
物化视图需要刷新策略；不是实时结果。

数据工程

Pipeline

多步骤、有依赖关系的数据任务流程。

详细解释
Pipeline 用于把多个查询、转换或处理步骤串起来，适合构建更完整的数据处理链路。
适合用法
适合自动化数据处理、定时刷新、多阶段指标生成。
注意点
需要明确失败重试、依赖关系和下游消费机制。

数据工程

ETL

Extract、Transform、Load，先抽取，再转换，最后写入目标库。

详细解释
ETL 是传统数据管道模式，适合把外部数据经过清洗后进入数据库。
适合用法
适合多服务商抓取、统一结构化、长期留存。
注意点
转换逻辑集中在数据管道，维护成本会随数据源增多上升。

数据工程

ELT

Extract、Load、Transform，先入库，再在数仓内转换。

详细解释
ELT 适合现代数据仓库，先保存原始数据，再用 SQL/dbt 做转换建模。
适合用法
适合保留原始响应、后续反复建模。
注意点
需要较好的数仓治理和成本控制。

AI Agent

MCP

Model Context Protocol，让 Agent 以标准工具协议连接 Dune。

详细解释
MCP 可以让 Codex、Cursor、Claude、OpenClaw 等工具型 Agent 调用 Dune 能力，而不是只靠自然语言复制粘贴。
适合用法
适合让 Agent 搜索表、执行查询、读取结果、生成分析。
注意点
必须限制权限、额度和可执行查询范围。

AI Agent

Agent Skills

可安装到 Agent 环境中的技能包，封装 Dune / Sim 的调用方法和提示规范。

详细解释
Skills 的价值是把 API 调用、使用规范和任务模板打包给 Agent，使其按固定方式使用 Dune。
适合用法
适合 Codex、Cursor、Claude Code、OpenClaw 的链上数据工具化。

Skill 不是安全边界；真正权限仍应由 API Key、代理服务和后端策略控制。

分页读取大结果集，避免单次返回过大。

详细解释

API / 结果读取

Partial Results

超大结果可能只返回部分数据，需要显式处理不完整结果。

详细解释
Partial Results 表示当前结果并非完整数据集，系统设计时要把它作为异常或降级状态处理。
适合用法
适合在 UI 中提示“结果被截断”，或在后端重新分片查询。

不能把 partial results 当成完整事实写入长期记忆。

链上数据

链上数据

Indexer

链上索引器，把区块、日志、交易和合约事件解析成可查询数据。

详细解释
Indexer 是链上应用的基础设施。自建 indexer 控制力强，但开发和维护成本高。Dune/Sim 本质上帮你承担了部分索引工作。
适合用法
适合高实时、自定义合约事件、强一致性业务。
注意点
Dune 适合分析和应用数据源，但不是所有场景都能替代自建 indexer。
返回 index 术语索引
链上数据

ABI

Application Binary Interface，智能合约函数和事件的接口描述。

详细解释
ABI 用来解码链上合约调用和事件日志。没有 ABI 时，链上原始数据难以直接读懂。
适合用法
适合解析合约事件、函数调用、Token 标准接口。
注意点
合约升级、代理合约和非标准事件会增加解析难度。

计费 / 额度

计费 / Dune

Credit

Dune Data Hub / Data API 的主要用量单位。

详细解释
执行查询、读取结果、导出数据、写入数据、刷新物化视图、通过部分连接器运行查询时，都会消耗计划内 Credits 或额外 Credits。
适合用法
用于估算 Dune SQL 分析、API 抓取、ETL 和自动化报告成本。
注意点
Credit 不等于固定请求次数。查询越复杂、扫描越多、运行越久，消耗越高。
返回定价与额度
计费 / 查询执行

Compute Credit

按实际计算资源扣减的 Credits。

详细解释
Compute Credit 与 CPU-seconds、网络资源、查询复杂度、执行时间和所选执行引擎有关。官方不公开单次查询固定公式。
适合用法
适合评估 SQL 查询、定时任务、物化视图刷新和 Connector 查询成本。
注意点
生产环境应为查询设置成本上限，并优先复用缓存结果或物化视图。
返回定价与额度
计费 / Sim

Compute Unit / CU

Sim API 的独立用量单位。

详细解释
Sim 按 API 调用消耗 CU。简单请求通常 1 CU，多链请求会按处理链数量消耗多个 CU。
适合用法
适合估算钱包余额、交易记录、NFT、Token Holder 等实时接口成本。
注意点
务必显式传 chain_ids，避免默认链集合变化导致 CU 消耗不可控。
返回定价与额度
计费 / 超额

Extra Credit

超过月度计划额度后继续使用产生的额外 Credits。

详细解释
当计划内 Credits 用完，如果允许超额使用，会按对应计划的额外 Credit 价格继续计费。
适合用法
适合偶发大查询或短期流量波动。
注意点
需要设置月度上限。若不想产生超额费用，应把额外 Credit 上限设为 0。
返回定价与额度
查询执行

Performance Tier

Dune 查询执行层级，常见为 Small、Medium、Large。

详细解释
不同 tier 分配不同计算资源。Small 成本低但资源少，Medium 适合常规查询，Large 适合复杂查询但通常消耗更多。
适合用法
根据查询复杂度和时效要求选择执行层级。
注意点
不要默认把所有任务放到 Large。应先优化 SQL，再提升 tier。
返回定价与额度
额度 / 限制

Quota

平台对用量、存储、并发、频率或功能数量设置的限制。

详细解释
Dune 的 Quota 包括每月 Credits、存储容量、Webhook 数量、API Rate Limit、私有查询数；Sim 的 Quota 包括 CU 和 RPS。
适合用法
用于设计限流、任务调度、用户分层和成本预算。
注意点
Quota 到达上限后可能失败、排队或产生额外费用，取决于具体计划和设置。
