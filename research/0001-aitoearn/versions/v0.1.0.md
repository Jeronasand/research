## Scope

Initial public-source research snapshot for AiToEarn, based on the China site, international site, official docs, GitHub repository, changelog, pricing docs, and status page accessed on 2026-05-11.

## Findings

AiToEarn is an open-source, AI-powered social media and content monetization platform. It combines a creator promotion marketplace with publishing, engagement, creation, and analytics workflows.

The China public site foregrounds creator earning through promotion tasks. Observed tasks include a pinned Jimeng CPE campaign, Xiaohongshu CPE tasks, Douyin-related historical tasks, and low-value interaction tasks. The international site mirrors the marketplace in USD and also exposes creator and advertiser roles.

The official product narrative centers on four pillars: Monetize, Publish, Engage, and Create. Monetize covers task-based creator earning and advertiser promotion tasks; Publish covers multi-platform scheduling and distribution; Engage covers AI-assisted replies, comment search, and social interaction; Create covers AI-generated copy, image, and video content.

Public docs and GitHub suggest a technically open posture: MIT-licensed repository, Docker Compose deployment, source development path, MCP/SSE endpoints, OpenClaw plugin, and API-key based integration. Self-hosting appears possible but depends on OAuth/channel setup or relay usage.

Release and operations evidence is mixed. The docs changelog shows v2.4.0 dated 2026-04-29, while the GitHub repository release panel observed publicly showed v2.1.0 as latest on 2026-03-28. The status page showed active issues and low displayed uptime, so SaaS reliability needs direct verification.

Pricing docs show Free/Plus plans and credit pricing, but the pricing pages are dated 2025-11. Treat pricing as stale until rechecked in-app or through current billing pages.

## Open Questions

- How robust is task validation against fake engagement and duplicate submissions?
- What is the full creator settlement and withdrawal flow?
- What is the advertiser refund and appeal process?
- Which social-platform automation behaviors are compliant and durable?
- Are GitHub releases, docs changelog, and in-app versions intentionally different release channels?
- Is the official status page accurately reflecting current production availability?
