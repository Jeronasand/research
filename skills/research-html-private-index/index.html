Research HTML Private Index

本 skill 约束 research 私有仓库的 HTML 调研文档、HTML skill、私有索引 JSON 和 OSS 双桶预览流程。

目录规则

  • 外部来源的 Web、HTML 或完整网页文件夹先放到 temptodo/，它是临时收件区，不是正式内容目录。
  • 待调研卡片维护在 research/pending/tasks.json，字段规范维护在 research/pending/tasks.schema.json。
  • 调研中 Web 静态包放到 research/in-progress/<topic>/index.html，资源保留在同一个目录树内。
  • 已完结调研 Web 静态包放到 research/completed/<topic>/index.html，资源保留在同一个目录树内。
  • 前端目录只展示 待调研、调研中、已完结调研 三类卡片，不展开包内所有 HTML。
  • skill 可以是 HTML；单页或带资源的 skill 都放到 skills/<skill-id>/index.html 所在目录内。

索引规则

私有索引文件是 research/private-index.json，必须根据真实目录结构和 tasks.json 生成，不维护虚假的手写条目。 temptodo/ 不进入索引；只有用户明确要求同步时，才把其中的单文件或文件夹分类到正式目录后再生成索引。 

node scripts/private-index.mjs
cd web
npm run sync:data

双桶边界

  • research-preview 只承载公开授权页和应用壳。
  • research-pages 承载 private manifest、pending task JSON、research package payload 和 private index。
  • 保持纯静态浏览器 AK/STS 访问，不引入后端网关。

打包上传

仓库根目录负责 OSS 打包和上传；授权桶与内容桶必须分开。

npm run package:oss
npm run upload:oss:dry-run
npm run upload:oss
  • dist/oss/auth-bucket/ 上传到 oss://research-preview/。
  • dist/oss/content-bucket/research-data/ 上传到 oss://research-pages/research-data/。
  • 只发布授权页时使用 npm run upload:oss -- --target auth。
  • 只发布内容数据时使用 npm run upload:oss -- --target content。
  • --delete 只能在 dry-run 后显式配合 --yes 使用。

完成检查

  • 没有把 temptodo/ 中未同步的文件直接发布或索引。
  • 新增 HTML 文档或 skill 后，已刷新 research/private-index.json。
  • 需要上传私有数据前，已生成 web/research-data/manifest.json。
  • 没有提交 AK/SK、STS token、签名 URL 或 session dump。
