Capability Map

先分能力，再选命令

`ossutil` 不只是上传工具。它同时覆盖配置、传输、检查、对象管理、桶管理、诊断和低层 API。这个 skill 的核心是让 AI 先识别风险，再组合命令。

Identity 配置和身份

管理配置文件、profile、AK/SK、STS、ECS RamRole、匿名模式和 region endpoint。

configversion
Transfer 上传、下载、复制

本地和 OSS 之间传文件，或在同区域 OSS bucket/prefix 之间复制对象。

cpappend
Mirror 目录同步

用递归遍历同步源和目标；只有明确需要镜像一致时才开启删除目标多余对象。

sync--delete--backup-dir
Inspect 检查对象和桶

列目录、看对象元信息、查容量、读取对象内容、计算 hash，适合发布后验证。

lsstatducathash
Admin 对象和桶管理

创建 bucket、目录占位对象，删除对象或桶，设置对象属性，恢复归档对象。

mbmkdirrmset-propsrestore
Advanced 诊断和低层 API

生成签名 URL、测上传下载链路，或直接调用 OSS API 和向量 API 管理高级配置。

presignprobeapivectors-api

Decision Matrix

场景到命令

这里列的是决策规则，不是命令大全。AI 应该先问清 source、destination、region、profile、权限和是否允许删除，再执行。

Static Publish 发布 dist 到 OSS

先查 bucket，再 dry-run 上传，确认范围后执行真实上传；静态站点发布默认排除 `.DS_Store`。

ossutil cp ./dist oss://<bucket>/<prefix>/ -r -n -f
Mirror 严格镜像同步

`sync --delete` 会删除目标端多余对象。只有目标 prefix 明确且 dry-run 结果符合预期时才允许执行。

ossutil sync ./dist oss://<bucket>/<prefix>/ -n --delete
Inspect 检查线上对象

发布后用 `stat` 查关键对象的 Content-Type、长度、Last-Modified；用 `ls` 限制数量查看 prefix。

ossutil stat oss://<bucket>/index.html
Share 临时分享对象

单对象临时访问优先用签名 URL，不为了分享一个文件而改 bucket ACL。

ossutil presign oss://<bucket>/<object> --expires 3600
Low-level API CORS / Website / Policy

高级 bucket 配置走 `ossutil api`，请求体文件必须进 review；涉及 public access 的变更必须单独确认。

ossutil api put-bucket-cors --bucket <bucket> --cors-configuration file://cors.json

Safety Flow

危险操作必须有确认点

删除、覆盖、权限变更和 bucket 级 API 都可能影响线上资产。这个 skill 默认把 dry-run、范围说明和发布后验证纳入输出。

01 识别目标

确认本地路径、OSS bucket、prefix、region、profile 和权限模式。

02 选命令

传输用 cp，同步用 sync，检查用 ls/stat，配置用 api。

03 预演

新目标、删除、覆盖、批量操作先加 `-n` dry-run。

04 执行

确认范围后执行真实命令，不在日志里输出密钥。

05 验证

用 stat、ls、curl 或业务 URL 验证对象和线上访问。
