OSS Dual Bucket Access

一个公开 Web 桶只负责登录和授权入口。用户在入口中提供 OSS AK/SK 或 STS 临时凭证后，页面用 key 授权访问其他私有资产桶或资产 prefix；公开桶不保存私有数据、资产内容和密钥。

项目边界说明：该页面是 research 调研仓库的本地副本与展示页，强调 “登录授权桶（public） + private 资产桶（private）”的双桶模型。 “skillgenome-pages” 仍可作为上游参考来源，但本页的发布与调试以本仓库为准，不需要在 skillgenome-pages 里重新运行构建发布。

公开桶只负责登录授权

公开 Web 桶只放登录页、授权 UI、JS、CSS 和 manifest。Private 资产桶保持私有 ACL，授权后的访问由浏览器中的 OSS SDK 使用 AccessKeyId、AccessKeySecret 和可选 securityToken 对目标资产桶发起签名请求。

长期 AK/SK 直接进入浏览器有泄露风险。必须优先使用 RAM 最小权限、只读前缀、短周期凭证；多人产品化时建议改为 STS 临时凭证或后端签名。

login web bucket private asset bucket AK/SK or STS CORS boundary
用户浏览器 打开公开站点，输入 AK/SK 或 STS token。
1
Login Web Bucket 公开读，只托管登录和授权入口。
OSS SDK 在浏览器内签名请求，不把密钥写入页面源码。
2
Private Asset Bucket 私有 ACL，通过 CORS + RAM 权限读取授权资产。
校验 匿名访问必须失败，授权访问只覆盖允许目录。
3
资产视图 只渲染已授权资产，不暴露 AK/SK，不写入 localStorage。

桶职责边界

组件 访问级别 职责
login web bucket public-read / CDN public 只托管登录/授权入口、JS、CSS 和 manifest，不保存密钥、private 数据或已授权资产内容。
private asset bucket private 保存 JSON、Markdown、媒体、导出文件或项目资产；匿名访问应返回 AccessDenied。
credential source 用户输入 AK/SK 或 STS 浏览器当前会话内使用，推荐 RAM 子账号和 prefix 级权限；生产多人场景优先 STS。

权限和跨域

RAM 最小权限

只给 private asset bucket 的指定 prefix 授权。只读模式需要 `oss:ListObjects` 和 `oss:GetObject`；上传再增加 `oss:PutObject`。

CORS 指定来源

private asset bucket 只允许公开登录 Web 域名和本地开发域名。直接 file:// 打开时浏览器来源是 null，本地调试需额外放行 null（建议仅临时使用）。不要把生产资产桶配置成所有来源可访问。

密钥不落盘

AK/SK 不进入 Git、公开登录桶、页面 HTML 或 localStorage。需要持久会话时用 STS 或服务端会话。

实现片段

          

          
const client = new OSS({
  region: "oss-cn-shenzhen",
  bucket: "research-pages",
  accessKeyId: userInput.accessKeyId,
  accessKeySecret: userInput.accessKeySecret,
  stsToken: userInput.securityToken // optional
});

const result = await client.list({
  prefix: "data/",
  "max-keys": 100
});

验证清单

        
    
          
  • 公开登录 Web 桶的 `index.html`、JS、CSS、manifest 可匿名访问。
    • 
          
  • Private 资产桶匿名访问返回 `AccessDenied`，不能因为 CORS 配置变成公开数据源。
    • 
          
  • Private 资产桶 CORS 只允许公开登录 Web 域名和本地开发域名，headers 包含 OSS 签名相关字段。
    • 
          
  • AK/SK 或 STS token 只能访问允许 prefix，不能越权读取整个桶。
    • 
          
  • 代码仓库、dist 和公开登录 Web 桶中不包含 AccessKeySecret、STS token 或 session dump。
