Dune 数据源提供商调研

Dune 的定位不是“普通行情 API”，而是一个面向 Web3 的链上数据平台：链上数据仓库 + DuneSQL 查询引擎 + Dashboard + Data API + MCP / Agent Skill。对 AI Agent、Codex、Trading Bot、链上监控和 Web3 数据聚合来说，它最有价值的部分是“可被 AI 消费的结构化链上数据层”。

100+
官方文档描述覆盖 100+ chains / blockchain datasets
API + MCP
既支持传统 API，也支持 Agent 直接接入
SQL First
核心能力围绕 DuneSQL 与 Query 资产化

一句话结论

Dune 适合作为链上数据研究、指标计算、Agent 数据发现、Webhook 通知和 Dashboard 的数据底座；不适合作为传统股票/外汇行情源，也不适合作为低延迟交易执行或 OrderBook 级实时撮合数据源。

对你的方向：如果是 AI Pulse / SkillGenome / Codex Trading Bot，Dune 更适合做“链上数据 Skill + MCP 数据源 + 定时指标计算 + 触发通知”，而不是每次让 Agent 临时扫大表。

核心能力

  • 查询链上原始表、decoded 表、curated 表。
  • 用 DuneSQL 生成指标和分析结果。
  • 通过 API 执行 Query、读取结果、管理 Query。
  • 通过 MCP 让 AI Agent 发现数据集、写 SQL、执行查询、管理可视化和 Dashboard。
  • 上传自己的 CSV/NDJSON 数据，与链上数据做 Join。
  • 使用 Materialized Views 和 Pipelines 降低重复计算成本。
  • 通过 Schedule + Webhook 触发外部通知。

优先研究顺序

  1. Data API：执行 Query / 获取结果。
  2. MCP：给 Codex、Claude Code、OpenCode、Cursor 使用。
  3. DuneSQL：表结构、性能、成本优化。
  4. Materialized Views：降低重复查询成本。
  5. Uploads：写入自己的策略/标签/事件数据。
  6. Webhooks：连接 AI Pulse / 手机通知 / Slack。
