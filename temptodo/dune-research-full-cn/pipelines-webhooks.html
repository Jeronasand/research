Pipelines / Webhooks

Pipelines 用来编排多个 Query 和 Materialized View 刷新；Webhooks 用来把 Query schedule 后的结果推给外部系统。

Pipelines

  • 将多个 Query 串成一个有依赖关系的工作流。
  • 可包含 Materialized View 刷新。
  • 可查看 pipeline execution status 和每个 node 状态。
  • 适合复杂日报、指标构建、分阶段数据处理。

Webhooks

  • 在 Query 编辑器中点击 schedule 设置定时执行。
  • 可填 webhook URL。
  • Query 执行后把结果和状态推给外部服务。
  • 适合 Slack、飞书、手机推送、AI Pulse。

Webhook 示例 payload 结构

{
  "message": "Query xxx was submitted for execution ...",
  "query_result": {
    "execution_id": "...",
    "query_id": 3106864,
    "state": "QUERY_STATE_COMPLETED",
    "submitted_at": "2024-01-17T04:18:49Z"
  }
}

和 AI Pulse 的推荐结合

Dune Query Schedule
  ↓ Webhook
本地 ai-pulse-server
  ↓ 判断 query_result.state / rows
事件规则引擎
  ↓
Flutter Push / Telegram / Slack / 飞书 / 本地通知

注意点

  • Webhook 是从 Dune 主动推送到你的服务，因此本地服务要么公网可访问，要么通过 tunnel / relay。
  • 如果只跑本地 Docker，可用 Cloudflare Tunnel / ngrok / 自建中转。
  • Webhook 接口需要验签或至少使用随机 secret path，避免被伪造触发。
