Dune MCP / AI Agent 能力

Dune 官方 MCP Server 让 AI Agent 直接接入 Dune 工作流：发现数据集、生成 SQL、执行查询、检查结果、管理可视化、构建 Dashboard。

连接信息

项目内容
MCP Server URLhttps://api.dune.com/mcp/v1
认证方式OAuth 2.0 或 Dune API Key
Header Authx-dune-api-key: <dune-api-key>
Query Auth?api_key=<dune_api_key>
适配对象Claude Code、Claude Desktop、OpenCode、Codex、Cursor 等

MCP 能力

  • Discover datasets：根据问题找到相关表。
  • Create SQL：生成 DuneSQL。
  • Execute SQL / Query：执行查询并返回结果。
  • Inspect result：检查执行状态和输出。
  • Visualization management：生成或管理可视化。
  • Dashboard management：构建 Dashboard。

Codex 接入示例

# OAuth 方式，适合有浏览器环境的 Agent
codex mcp add dune https://api.dune.com/mcp/v1

# API Key 方式，适合无浏览器、本地服务、Docker 环境
# 具体命令以 Codex 当前版本为准，核心是给 MCP 请求加：
x-dune-api-key: YOUR_DUNE_API_KEY

注意：不同 Agent 客户端对 Header 配置支持不同。如果不能配置 Header，可以使用 URL query 参数方式，但安全性较差，不建议在共享日志环境使用。

风险控制

  • MCP Agent 使用的权限等同于 API Key 权限，要按最小权限原则配置。
  • 禁止让 Agent 无限制执行任意 SQL，尤其是大范围历史扫描。
  • 把常用任务固化成 Skill 或 Query 模板，减少动态 SQL。
  • Agent 每次执行前应估算：目标表、时间范围、是否需要 Materialized View。
