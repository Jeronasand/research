Dune 数据源提供商调研

定位：Web3 / On-chain Analytics 数据平台，适合链上数据查询、仪表盘、研究分析、AI Agent 调用、Trading Bot 的链上信号补充。

Crypto / On-chain SQL API MCP Dashboards 100+ Blockchains
综合评级
4.4 / 5

结论：强烈推荐作为链上数据分析与 AI Agent 数据源；不适合作为低延迟交易所行情源或传统股票行情源。

核心优势

链上数据覆盖广，支持 raw / decoded / curated 数据，SQL 查询生态成熟，公开 dashboard 与 query 资源丰富。

主要短板

非传统金融数据源；不提供 A 股 / 港股 / 美股行情。实时交易场景需要搭配交易所、行情商或节点服务。

AI 适配度

官方提供 Remote MCP，可用于 Codex / Claude / Cursor 等客户端进行表发现、SQL 查询、结果读取、可视化和 Dashboard 管理。

一、基础信息

名称Dune
官网https://dune.com/home
文档https://docs.dune.com/
产品定位All-in-one crypto / onchain data platform：SQL 查询、API、Datashare、Dashboard、MCP Agent 接入。
数据类型区块链 raw data、decoded contract events/calls、curated datasets、DEX trades、NFT trades、token transfers、balances、prices、stablecoins、CEX flows、bridges、lending、gas/fees、labels 等。
目标用户Web3 数据分析师、研究员、协议团队、基金/投研、数据工程、AI Agent、链上监控系统。

二、数据覆盖能力

数据项支持情况说明
Crypto 链上数据强支持覆盖 100+ blockchains，包含 raw / decoded / curated 三层数据。
DEX Trades强支持适合做协议交易量、用户行为、链上交易路径分析。
Token Prices支持价格数据覆盖 70+ chains，提供多粒度价格表。
NFT / Stablecoin / Bridge / Lending支持有 curated datasets，可减少重复清洗。
CEX orderbook / Tick 行情不适合Dune 不是交易所低延迟行情源。
A 股 / 港股 / 美股不支持应搭配 Polygon、Finnhub、Twelve Data、AkShare 等。
新闻 / 社媒舆情不作为主项需要搭配新闻 API、Twitter/X 数据源、RSS 或搜索源。

三、接口与开发能力

能力支持情况备注
REST API支持可执行 saved query、raw SQL、获取 execution status/result、CSV 结果等。
SQL Execute API支持POST https://api.dune.com/api/v1/sql/execute
Webhooks支持适合查询执行结果、监控/通知类流程。
SDK支持官方文档列出 Python、TypeScript、Go SDK。
MCP强支持官方 Remote MCP：https://api.dune.com/mcp/v1
Datashare / BI Connector支持适合企业 BI、数据仓库或更稳定的数据管道。
WebSocket非核心Dune 更偏查询与分析，不是 tick 级推送行情源。

API 示例

curl -X POST "https://api.dune.com/api/v1/sql/execute" \
  -H "Content-Type: application/json" \
  -H "X-Dune-Api-Key: YOUR_API_KEY" \
  -d '{
    "sql": "SELECT * FROM dex.trades WHERE block_time > now() - interval '1' day LIMIT 10",
    "performance": "medium"
  }'

Codex MCP 配置

codex mcp add dune --url "https://api.dune.com/mcp/v1"
codex mcp login dune

# API Key 方式
codex mcp add dune_prod --url "https://api.dune.com/mcp/v1?api_key=<dune_api_key>"

四、AI Agent / Codex 使用评估

维度评分说明
Tool Calling 友好度5 / 5官方 MCP 提供 searchTables、executeQuery、getExecutionResults、generateVisualization、createDashboard 等工具。
结构化输出4.5 / 5SQL 查询结果天然结构化，适合进入 LLM 分析链路。
事件通知4 / 5API 文档包含 Webhooks，可用于执行完成或监控任务回调；具体事件粒度需要实测。
Trading Bot 适配3.5 / 5适合做链上信号、地址监控、协议流量分析；不适合直接替代实时交易行情源。
本地服务调用4 / 5可通过 MCP / API 在本地 Agent 中接入，但 API Key 与查询成本需要控制。

五、风险与注意事项

风险等级说明
成本不可控中API 查询按 credits 计费，失败执行也可能消耗 credits。需要做 query scan 预估、缓存与限流。
实时性不足中适合分析，不适合毫秒级/秒级行情策略。
SQL 复杂度中复杂链上查询容易扫描过大，需优化分区、时间范围、字段选择。
数据范围偏 Web3高不能覆盖传统金融市场，需要组合其它股票/宏观/新闻数据源。

六、适合接入架构

AI Agent / Codex
   ↓ MCP / REST API
Dune MCP or Dune API
   ↓ SQL Query / Saved Query
Dune Data Warehouse
   ↓ Structured Result
Local Service / Trading Bot / Dashboard / Notification System

推荐做法

  • 高频任务不要直接让 Agent 随意生成 SQL；先沉淀 saved query。
  • 对常用 query 做缓存，例如 Redis / Postgres，避免重复消耗 credits。
  • 把 Dune 用作“链上分析源”，把 Binance / OKX / Polygon / Finnhub 用作“实时行情源”。
  • Codex 接 MCP 时建议提高 tool timeout，长查询可能超过默认 60 秒。

七、最终结论

是否推荐推荐
推荐用途链上数据研究、协议分析、钱包/地址监控、链上指标、Web3 dashboard、AI Agent 数据查询。
不推荐用途股票数据、低延迟 orderbook、交易所实时成交流、新闻舆情主数据源。
组合建议Dune + Binance/OKX 行情 + CoinGecko/CoinMarketCap 市值 + 新闻/RSS + 股票 API。
